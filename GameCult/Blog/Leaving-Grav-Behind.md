# Leaving Grav Behind

The old `gamecult.org` site proved something useful: publishing from Git was already better than editing through a mystery box.

What it never solved was ergonomics. Grav content was technically editable in Git, but the shape of the files still felt like they belonged to the CMS first and the writer second. The Aetheria Quartz workflow showed the opposite: content-first Markdown, local preview, GitHub Pages deployment, and a front end we could actually shape around the project.

That is the reason for the move. This is less about static sites being fashionable and more about choosing a publishing model that matches how GameCult already works.
